Linux常用操作指令总结

Posted on 2019-10-15

网上相关的文章有很多，但是比较松散，这里会作为一个整体上的总结，希望能加深自己的记忆（以 Debian 系统为例）。

信息查询

这一类型指令包含查询当前 Linux 系统的一些基本信息，例如内存、CPU、磁盘等等，有助于对服务器的原始状态有一个初步了解。一些基本指令是基于 dmidecode 工具的，如果未找到指令，使用 sudo apt-get install dmidecode 安装。

服务器

  1. 查看服务器品牌型号信息

    dmidecode | grep -A2 "System Information" | grep "Product Name"

内存

  1. 查看每个插槽的内存大小

    dmidecode | grep -A5 "Memory Device" | grep Size

  2. 查看每个插槽的内存频率

    dmidecode | grep -A16 "Memory Device" | grep "Speed"

CPU

  1. 查看 CPU 个数、核心数

    • 物理 CPU 个数：cat /proc/cpuinfo | grep "physical id" | sort | uniq | wc -l
    • 逻辑 CPU 个数：cat /proc/cpuinfo| grep "processor"| wc -l
    • 查看单个 CPU 核心数：cat /proc/cpuinfo| grep "cpu cores"| uniq

    另外要关注的是如下两个关于 CPU 的计算公式

    • 总核心数 = 物理 CPU 个数 * 单个 CPU 核心数
    • 逻辑 CPU 个数 = 物理 CPU 个数 单个 CPU 核心数 超线程数

  2. 查看 CPU 型号

    cat /proc/cpuinfo | grep name | cut -f2 -d: | uniq -c

    PS: 可以留意下这里 cut 分割字符串的用法，比 awk 简单直接一些。

  3. 查看 CPU 频率

    cat /proc/cpuinfo | grep "cpu MHz"

磁盘

查看磁盘的场景应该是最多的，例如我们需要确认磁盘状态、磁盘类型，从而进行一些维护处理，都需要用到磁盘查看的相关指令。

  1. 确认磁盘是否为 SSD

    • 利用 block layer 特性判断：我们以 sda 为例，通过 cat /sys/block/sda/queue/rotational 指令查看返回，如果为 0 则为 SSD 盘，返回 1 则为普通 HDD。然而，这个方式也有不准确的可能性，具体可以参考 此链接
    • 利用 lsblk 指令判断：其实跟前面的方法类似，使用 lsblk -d -o name,rota 指令用于一次性显示各个磁盘是否为 rotational。
    • 使用 smartctl 指令判断：此指令包含在 smartmontools 包中，使用 smartctl -i /dev/sda 查看磁盘信息（做了 raid 的盘目前也不是很清楚怎么进行查询判断）。另外关于 SMART 工具，可以参考 此文章

    所以，上述方法最准确的使用场景应该是在没有做 Raid 的情况下。如果在做了 Raid 的情况下，我们可以使用以下方法。

    • 利用 hwtool 指令判断：使用 hwtool -m raid -d 指令的输出，即可查看磁盘信息，以及最重要的 磁盘状态 也可以使用此命令查询到
    • 利用 MegaCli 指令判断：使用 /usr/local/hwtool/tool/MegaCli PDList aAll 指令的输出，即可查看物理盘的具体信息，同样，磁盘状态 也可以使用此命令查询到
